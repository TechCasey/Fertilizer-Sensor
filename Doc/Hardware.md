# 硬件连接详细说明

本文档描述GD32F4-Fertilizer-Sensor项目的硬件连接方法。

---

## 🔧 所需硬件

### 必需
- GD32F407开发板（或其他GD32F4系列）
- 光电传感器模块（双通道：可见光 + 近红外）
- USB-TTL串口模块（用于调试输出）
- 杜邦线若干
- 电源（3.3V稳定电源）

### 可选（用于CAN通信测试）
- TJA1050 CAN收发器模块（或SN65HVD230）
- CAN分析仪（如PCAN-USB）
- 120Ω终端电阻（2个）

---

## 🔌 硬件连接

### 1. 光电传感器 ↔ GD32（ADC接口）

光电传感器输出模拟电压信号，连接到GD32的ADC输入通道。

| 光电传感器引脚 | 说明 | GD32引脚 | 备注 |
|---------------|------|----------|------|
| VCC | 电源正（3.3V/5V） | 3.3V/5V | 根据传感器规格 |
| GND | 电源地 | GND | - |
| OUT1 | 信号输出1（可见光） | PA0（ADC0_CH0） | ADC采样通道0 |
| OUT2 | 信号输出2（近红外） | PA1（ADC0_CH1） | ADC采样通道1 |
| LED_CTRL | LED光强控制输入 | PA4（DAC0_OUT） | DAC输出控制LED |
| LED_GND | LED电源地 | GND | - |

**ADC采样说明**:
- ADC0_CH0（PA0）：采样可见光通道输出信号
- ADC0_CH1（PA1）：采样近红外通道输出信号
- ADC分辨率：12位（0-4095）
- 采样模式：DMA连续采样

**DAC输出说明**:
- DAC0_OUT（PA4）：输出0-3.3V模拟电压，用于控制LED光强
- DAC分辨率：12位（0-4095对应0-3.3V）

---

### 2. CAN总线 ↔ GD32

GD32内置CAN控制器，但需要外接CAN收发器（如TJA1050）才能连接到CAN总线。

| TJA1050引脚 | 说明 | GD32引脚 | 备注 |
|-------------|------|----------|------|
| VCC | 电源正（3.3V或5V） | 3.3V/5V | 取决于模块 |
| GND | 电源地 | GND | - |
| RXD | 接收数据 | PB8（CAN0_RX） | CAN接收引脚 |
| TXD | 发送数据 | PB9（CAN0_TX） | CAN发送引脚 |
| CANH | CAN高电平线 | CAN总线H | 需要120Ω终端电阻 |
| CANL | CAN低电平线 | CAN总线L | 需要120Ω终端电阻 |

**CAN总线终端电阻**:
- CAN总线两端各需要1个120Ω终端电阻
- 如果只有两个节点，每个节点的CANH和CANL之间并联1个120Ω电阻

---

### 3. 调试串口 ↔ GD32

用于打印调试信息（printf输出）。

| USB-TTL模块 | GD32引脚 | 说明 |
|-------------|----------|------|
| VCC | 不连接 | 无需供电 |
| GND | GND | 共地 |
| TXD | PB10（USART2_RX） | 模块发送 → GD32接收 |
| RXD | PB11（USART2_TX） | 模块接收 ← GD32发送 |

**注意**: GD32的USART2_TX（PB11）连接到USB-TTL的RXD，交叉连接！

---

### 4. LED指示灯 ↔ GD32

用于指示系统运行状态。

| LED | GD32引脚 | 说明 |
|-----|----------|------|
| LED1 | PC0 | 系统运行指示（每秒闪烁） |
| LED2 | PC1 | 错误指示（初始化失败时闪烁） |
| LED3 | PC2 | CAN通信指示（每次发送时闪烁） |

**连接方法**:
- LED正极（长脚）→ 220Ω限流电阻 → GD32引脚
- LED负极（短脚）→ GND

---

## 🔋 电源说明

### 3.3V供电
- GD32开发板通常通过USB供电（5V），板载3.3V稳压芯片
- 光电传感器和TJA1050可以使用GD32开发板的3.3V输出

### 5V供电
- 如果光电传感器需要5V供电，可以连接到GD32开发板的5V输出
- **注意**: GD32的GPIO是3.3V电平，ADC输入最大不能超过3.3V

**推荐**: 使用3.3V版本的光电传感器模块，避免电平转换问题。

---

## 🔍 硬件检查清单

在通电前，请检查以下项目：

- [ ] **电源**: 所有模块都连接到正确的电源电压（3.3V或5V）
- [ ] **接地**: 所有模块的GND都连接到公共地
- [ ] **ADC输入范围**: ADC输入电压不超过3.3V（可调节传感器增益）
- [ ] **CAN终端电阻**: CAN总线两端都有120Ω终端电阻
- [ ] **LED限流**: LED都串联了220Ω限流电阻
- [ ] **串口交叉**: TXD和RXD正确交叉连接
- [ ] **无短路**: 用万用表检查电源和地之间没有短路

## 📏 硬件连接示例（完整系统）

```
[GD32F407开发板]
    |
    |--- PB10（USART2_RX）-- [USB-TTL模块] ---- PC（调试输出）
    |--- PB11（USART2_TX）-+
    |--- GND ----------------+
    |
    |--- PA0（ADC0_CH0）---- [光电传感器]（肥料检测）
    |--- PA1（ADC0_CH1）---+
    |--- PA4（DAC0_OUT）--- LED_CTRL（控制LED光强）
    |--- 3.3V -------------- VCC
    |--- GND ---------------- GND
    |
    |--- PB8（CAN0_RX）----- [TJA1050模块] ---- [CAN总线]
    |--- PB9（CAN0_TX）----+
    |--- 3.3V -------------- VCC
    |--- GND ---------------- GND
    |
    |--- PC0 ---- 220Ω ---- LED1（运行指示）
    |--- PC1 ---- 220Ω ---- LED2（错误指示）
    |--- PC2 ---- 220Ω ---- LED3（CAN指示）
```

## 🐛 硬件故障排查

### 问题1: ADC采样值始终为0
**可能原因**:
- 传感器未正确连接
- ADC引脚配置错误
- 传感器未上电

**解决方法**:
1. 用万用表测量传感器OUT1/OUT2引脚电压（应为0-3.3V）
2. 检查GD32的ADC初始化代码
3. 确认传感器电源和地正确连接

### 问题2: DAC输出无电压
**可能原因**:
- DAC未初始化
- DAC输出引脚配置错误
- DAC数据寄存器未写入

**解决方法**:
1. 用万用表测量PA4引脚电压
2. 检查DAC初始化代码
3. 确认DAC数据寄存器已写入

### 问题3: CAN无法通信
**可能原因**:
- TJA1050未正确连接
- CAN总线终端电阻缺失
- CANH和CANL接反
- 波特率不匹配

**解决方法**:
1. 检查TJA1050的RXD和TXD是否正确连接到GD32
2. 在CAN总线两端各并联1个120Ω电阻
3. 确认所有节点的CAN波特率一致（默认500Kbps）

### 问题4: 串口无输出
**可能原因**:
- USB-TTL模块未正确连接
- 波特率不匹配
- GD32串口引脚配置错误

**解决方法**:
1. 检查USB-TTL模块的TXD和RXD是否交叉连接
2. 确认PC端串口助手波特率设置为115200
3. 用示波器观察PB11（USART2_TX）是否有数据输出
